# 给第二届天融信班学弟学妹们的实训经验分享

第二届天融信班的学弟学妹们，大家好。

我是第一届天融信班的一名学长。今天大家正式开班，我想结合过往实训经历，和大家分享一下在实训过程中可能会遇到的问题，以及一些值得提前关注的学习方向。

这次分享不会只是告诉大家“每天要完成作业”，而是想聊一聊：在这段实训时间里，怎样才能真正把每天学到的东西变成自己的能力，怎样利用 AI 提升学习和实践效率，以及后面准备简历、面试时应该注意什么。

---

## 一、AI 篇：不要只听概念，要真正用起来

这两年 AI 发展非常快，而且 AI 能力已经逐渐成为企业招聘时会关注的一部分。对于我们这个方向的同学来说，AI 不是一个“可有可无”的工具，而是一个可以显著提升效率的能力放大器。

### 1. 先搞懂 AI 相关概念

现在和 AI 相关的概念非常多，比如：

*   Prompt
    
*   Tools
    
*   Skills
    
*   RAG
    
*   Agent
    
*   MCP
    

这些词看起来很杂，但它们背后其实都对应着 AI 的不同能力和使用方式。

大家可以先问自己几个问题：

1.  这些概念自己有没有了解过？
    
2.  如果了解过，有没有真正动手用过？
    
3.  如果只是听说过，能不能用自己的话讲清楚它们是干什么的？
    
4.  能不能把这些能力用到自己的学习、作业和项目里？
    

如果只是停留在“听过”，其实价值不大。真正重要的是动手尝试。

### 2. AI Agent 能帮我们做什么？

AI Agent 已经不只是简单聊天了，它可以帮助我们完成很多实际任务，例如：

*   写代码
    
*   辅助渗透测试
    
*   制作 PPT
    
*   完成文档和作业
    
*   部署环境
    
*   排查问题
    
*   自动化处理重复任务
    

比如在现场可以演示一些实际场景：

*   起一个 Pikachu 靶场
    
*   起一个 DVWA 靶场
    
*   给虚拟机扩容磁盘
    
*   安装一个最近比较火的游戏《赵云与阿斗》
    

这些事情以前可能需要我们查很多教程、一步步试错。但在 AI Agent 的帮助下，只要你有基本概念，知道自己想做什么，它就能帮助你快速实现。

当然，AI 不是万能的。大家也需要了解它的能力边界：它可能会写错命令、误解环境、生成看似合理但实际有问题的方案。所以使用 AI 的时候，不能完全照抄，要学会验证、排错和复盘。

在现在这个 AI 时代，只要你有想法、有概念、有验证能力，很多事情就可以尝试做出来。

## 二、要舍得投入：工具会直接影响效率

很多同学可能会觉得 AI 会员很贵。比如 ChatGPT、Claude 这些工具，一个月会员可能要 140 元以上。

这确实不是一笔特别小的钱，但我想说的是：在模型迭代这么快的阶段，适当投入是值得的。

原因很简单：

1.  新模型能力提升非常快。
    
2.  你需要持续体验最新工具。
    
3.  你越早熟悉这些工具，越容易在学习和项目中形成优势。
    
4.  很多事情靠自己摸索可能要几个小时，但用好 AI 可能几十分钟就能解决。
    

我个人比较推荐大家至少体验 ChatGPT Plus。不是说它一定适合所有人，而是它确实能让你更早接触到比较新的 AI 能力。

## 三、AI Coding + Git：让想法真正变成项目

AI Coding 和 Git 是非常搭配的组合。

AI Coding 可以放大你的想象力，帮你更快地写代码、搭框架、改 bug、补功能。Git 则可以帮你记录每一次迭代，相当于给你一颗“后悔药”。

### 1. 为什么一定要学 Git？

因为你在写项目的时候一定会遇到这些情况：

*   改着改着项目跑不起来了
    
*   不知道是哪一步改坏了
    
*   想回到之前某个版本
    
*   想对比前后代码差异
    
*   想把项目发布到 GitHub 上
    

这些问题都可以通过 Git 更好地解决。

### 2. 实训期间可以做什么？

大家可以在实训期间尝试做一些 AI Coding 的小产物，例如：

*   自动化脚本
    
*   靶场部署脚本
    
*   安全工具小插件
    
*   漏洞复现环境
    
*   信息收集工具
    
*   学习笔记网站
    
*   简单的 Web 项目
    

完成之后，可以把项目发布到 GitHub 上，并把链接贴到自己的简历里。

这样做的好处是：面试官看到你的简历时，不只是看到一句“熟悉某某技术”，而是能看到你真的做过东西，有实际产物可以证明。

## 四、工欲善其事，必先利其器

学习安全、开发、运维这些方向，工具非常重要。工具选得好，效率会高很多。

### 1. 访问外网的能力

很多先进的工具、文章、文档和开源项目，都需要通过外网才能方便访问。

比如很多技术资料、GitHub 项目、英文文档、AI 工具，国内访问可能不稳定。这个问题如果解决不好，会严重影响学习效率。

所以大家需要尽早具备稳定获取外部技术资源的能力。

### 2. 关于电脑系统

如果条件允许，AI 时代我比较推荐使用 Mac。

原因是 Mac 基于 Unix 系统，很多开发、安全、AI 相关工具对 Unix-like 环境支持更好，安装和配置也会更顺滑。

相比之下，Windows 的 PowerShell 在某些场景下会比较折磨，尤其是涉及命令行引号、转义、路径兼容的时候，AI 也经常会被 PowerShell 的细节搞乱。

例如 Codex 自己都会觉得 PowerShell 的引号问题很烦。

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/eYVOL5jk2a2wLlpz/img/f2fae929-1a61-4831-827d-be7a6d45cf74.png)

当然，这不是说 Windows 不能用，而是说如果你以后经常折腾开发、安全工具、AI Coding、自动化脚本，Unix-like 环境会更省心。

## 五、每日作业篇：不要只完成任务，要学会拓展

实训期间每天都会有作业，但大家一定不要只把作业当成“交差任务”。

如果你只是按照老师给的步骤，把结果截图交上去，那你确实完成了作业，但不一定真正掌握了能力。

更好的方式是：把每天的作业当成一个探索未知事物的锚点。

### 1. 从 Windows Server 拓展到 Linux

比如在 Windows Server 阶段，大家可能会学习：

*   DNS
    
*   DHCP
    
*   Web 中间件
    
*   证书
    
*   域控相关内容
    

基地可能会教你如何在 Windows Server 上部署这些服务。

但你还需要进一步思考：

*   这些服务在 Linux 上怎么部署？
    
*   Linux 下有哪些常见实现？
    
*   配置文件在哪里？
    
*   服务怎么启动、停止、排错？
    
*   Windows 和 Linux 的实现有什么区别？
    

只有这样，你的理解才不会停留在某一个操作步骤上。

### 2. 从自签证书拓展到公网证书

再比如，基地可能会教大家如何在 Windows Server 中自签证书。

但你还可以继续思考：

*   如果给你一个公网 IP 和公网域名，你会申请证书吗？
    
*   你会使用 Let’s Encrypt 吗？
    
*   你知道证书签发、续期、域名解析之间的关系吗？
    
*   你知道 HTTP-01、DNS-01 验证方式吗？
    
*   你会给 Nginx 或 Apache 配 HTTPS 吗？
    

这些才是真实场景中经常会遇到的问题。

### 3. 从 phpStudy 拓展到手动部署环境

基地在让大家部署靶场的时候，为了省事，可能会让大家在现成系统里用 phpStudy 起靶场。

但你可以继续问自己：

*   能不能不用 phpStudy，自己手动安装 LAMP 环境？
    
*   能不能用 Nginx 来起 Web 服务？
    
*   能不能用 Docker 来部署靶场？
    
*   能不能把部署过程写成脚本？
    
*   能不能记录成一篇完整博客？
    

如果你只循规蹈矩，只完成手中的作业，那你和大多数人不会有什么明显差距。

真正能拉开差距的，是你能不能在作业基础上多问几个“为什么”和“还能不能换一种方式实现”。

保持好奇心，不要把作业只当作作业。

## 六、简历和面试篇：提前准备，不要临时抱佛脚

从 8 月开始，大家会陆陆续续准备简历并开始投递。

这个阶段非常重要，因为只有简历过关，你才会有面试机会；只有面试准备充分，你才有可能拿到 offer。

### 1. 面试前：先把简历准备好

一份还不错的简历，至少要做到以下几点：

#### 第一，排版简洁大气

简历不要花里胡哨，也不要堆满文字。重点是让面试官能快速看到你的优势。

你需要突出：

*   技术栈
    
*   项目经历
    
*   实训经历
    
*   安全能力
    
*   工具使用能力
    
*   实际产物链接
    

#### 第二，合理包装和提炼经历

简历上的经历要尽量基于真实学习、真实项目、真实实践来提炼。

如果某些经历还不够完整，可以适当包装表达方式，但不要写自己完全不懂、完全没做过的东西。

因为面试官很可能会追问：

*   这个项目你负责了哪一部分？
    
*   遇到了什么问题？
    
*   怎么解决的？
    
*   用到了哪些技术？
    
*   为什么这么设计？
    
*   有没有复现过程？
    
*   有没有部署记录？
    
*   有没有代码链接？
    

如果你答不上来，反而会暴露问题。

#### 第三，有实际产物就贴链接

如果你有 GitHub 项目、博客文章、靶场复现记录、工具脚本、在线文档，都可以贴到简历中。

实际产物比空泛描述更有说服力。

例如：

*   GitHub 项目链接
    
*   博客链接
    
*   在线文档链接
    
*   靶场搭建记录
    
*   漏洞复现文章
    
*   自动化工具代码
    

这些都可以成为面试中的加分项。

#### 第四，用 AI 反向准备面试问题

大家可以把自己的简历发给 AI，让它根据简历生成可能被问到的问题。

比如让 AI 帮你生成：

*   项目追问
    
*   技术细节问题
    
*   八股问题
    
*   场景题
    
*   简历漏洞点
    
*   面试官可能质疑的地方
    

然后你提前准备答案。

最基本的一点是：不要连自己简历上的内容都回答不出来。

### 2. 面试中：紧张是正常的

刚开始面试的时候，很多人都会紧张，这是正常现象。

不要因为第一次、第二次面试表现不好，就觉得自己不行。

面试能力本身也是需要练习的。你需要在一场又一场面试中积累经验。

### 3. 面试后：一定要复盘

每次面试后，都建议进行复盘。

可以记录：

*   哪些问题没答上来
    
*   哪些问题答得不流畅
    
*   哪些项目细节被问住了
    
*   哪些基础知识不扎实
    
*   面试官重点关注了什么
    
*   下次应该怎么回答
    

如果条件允许，可以在面试过程中给自己录音或录屏，方便后续复盘。当然，要注意不同平台和公司的规则，不要泄露敏感信息。

不要因为几场面试挂掉就灰心，也不要因此变得不自信。

挂面试是很正常的事情，重要的是每次面试之后有没有改进。

## 七、补充知识：基地不一定系统教，但你最好要会

有些内容基地可能不会系统地教，但对于后续就业和实际工作来说非常重要。

建议大家在实训期间或实训后，逐步补充这些知识。

### 1. Docker

Docker 是非常重要的基础能力。

你至少需要掌握：

*   镜像
    
*   容器
    
*   Dockerfile
    
*   docker-compose
    
*   端口映射
    
*   数据卷
    
*   网络
    
*   常见排错
    

很多靶场、工具、服务都可以用 Docker 快速部署。

### 2. Kubernetes

K8s 可以先不追求特别深入，但至少要了解基本概念：

*   Pod
    
*   Deployment
    
*   Service
    
*   Ingress
    
*   ConfigMap
    
*   Secret
    
*   Namespace
    

如果后面走云安全、运维安全、 DevSecOps 方向，K8s 是绕不开的。

### 3. 云基础设施

云相关知识也很重要，尤其是现在很多企业的业务都跑在云上。

建议大家了解：

*   VPC
    
*   ECS
    
*   EIP
    
*   SLB
    
*   OSS
    
*   SLS
    
*   安全组
    
*   IAM 权限
    
*   云服务器部署
    
*   云上日志和监控
    

这些内容对云安全、渗透测试、运维、安全运营都有帮助。

## 十、给基础不太好的同学

最后，想专门对基础不太好的同学说几句。

如果你在代码审计阶段开始觉得云里雾里，听不懂、跟不上，这是很正常的。不要因此直接放弃，也不要盲目硬冲后面的内容。

我的建议是：把重心放回前面的基础阶段。

你可以优先补这些内容：

*   Linux 基础命令
    
*   Web 基础
    
*   HTTP 协议
    
*   HTML、CSS、JavaScript 基础
    
*   PHP 或 Python 基础
    
*   MySQL 基础
    
*   常见 Web 漏洞原理
    
*   靶场部署
    
*   基础网络知识
    
*   Docker 基础
    

代码审计对基础要求比较高，如果前面的 Web、代码、漏洞原理不扎实，后面听起来就会很痛苦。

所以不要只盯着进度，而是要回头补基础。

基础打牢了，后面的内容才能真正听懂。

## 结语

实训这段时间其实很短，大家不要只想着完成每天的任务，也不要只把它当成一次培训。

更重要的是利用这段时间养成几个习惯：

1.  遇到问题主动探索。
    
2.  不满足于老师给的步骤。
    
3.  用 AI 提升效率，但不盲目相信 AI。
    
4.  把作业拓展成项目。
    
5.  把项目沉淀成作品。
    
6.  把作品放进简历。
    
7.  面试后持续复盘。
    
8.  不断补齐自己的基础短板。
    

只要你能持续保持好奇心，把每天学到的东西真正变成自己的能力，这段实训就不会白过。

希望大家都能在接下来的实训中有所收获，也希望大家后面都能拿到满意的 offer。
